Architecture Overview

Resource Details

N Virtual Network

  • Address space: 10.0.0.0/16
  • 5 subnets: web, app, db, kv, Bastion
  • NSG per subnet — deny-by-default
  • Web blocked from reaching DB directly

W Application Gateway

  • SKU: WAF_v2 (autoscale 1–3)
  • OWASP 3.2 rule set — Prevention mode
  • TLS termination via Key Vault cert
  • HTTP → HTTPS permanent redirect
  • Diagnostics to Log Analytics

C Linux VMSS

  • Ubuntu 22.04 LTS — Standard_B2s
  • SSH-only authentication (no passwords)
  • System-assigned Managed Identity
  • Autoscale: 2–5 instances (CPU thresholds)
  • Boot diagnostics + Log Analytics agent

D Azure SQL Database

  • Private Endpoint only — no public access
  • Minimum TLS 1.2 enforced
  • Entra ID (Azure AD) authentication
  • Zone-redundant (prod)
  • Short + long term backup retention
  • SQL audit logs → Log Analytics

K Key Vault

  • RBAC authorization (no access policies)
  • Purge protection + 90-day soft delete
  • Network ACLs: deny default, allow deployer + subnets
  • Deployer → KV Admin role
  • AppGW identity → Secrets User
  • VMSS identity → Secrets User

B Azure Bastion

  • Dedicated AzureBastionSubnet (/26)
  • Secure SSH to VMSS — no public IPs on VMs
  • Standard SKU with native client support
  • Full NSG: GatewayManager, LB, host-to-host

L Log Analytics

  • Central workspace: PerGB2018 SKU
  • Collects: AppGW access + WAF logs
  • Collects: SQL audit logs
  • Collects: VMSS boot diagnostics
  • 30-day retention
